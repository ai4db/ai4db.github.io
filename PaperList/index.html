1. Approximate Query Processing

  • DBEST: Revisiting approximate query processing engines with machine learning models (SIGMOD 2019)

  • Learned Approximate Query Processing: Make it Light, Accurate and Fast (CIDR 2021)

    DBEst的后续工作

    DBEst最新代码：https://github.com/qingzma/DBEstClient

  • Approximate Query Processing for Data Exploration using Deep Generative Models (ICDE 2020)

    使用VAE的方法

哈工大挂出来的两篇，似乎都还没中：

  • LAQP: Learning-based Approximate Query Processing (2020.3)
  • Approximate Query Processing for Group-By Queries based on Conditional Generative Models (2021.1)

2. Cardinality Estimation

  • Are We Ready For Learned Cardinality Estimation? (arXiv 2020)

    对近年基数估计论文进行了实验对比和讨论

    https://github.com/sfu-db/AreCELearnedYet

  • A Unified Deep Model of Learning from both Data and Queries for Cardinality Estimation (SIGMOD 2021)

    结合了data-driven和query-driven

1) Data-based

传统机器学习方法:

  • DeepDB: Learn from Data, not from Queries! (VLDB 2020)

    使用Sum-Product Networks

    https://github.com/DataManagementLab/deepdb-public

  • FLAT: Fast, Lightweight and Accurate Method for Cardinality Estimation (VLDB 2020)

    对Sum-Product Networks再次进行改进

  • An Approach Based on Bayesian Networks for Query Selectivity Estimation (DASFAA 2019)

    使用贝叶斯网络

  • BayesCard: A Unified Bayesian Framework for Cardinality Estimation (arXiv 2020)

    使用贝叶斯网络

深度学习方法：

  • Deep Unsupervised Cardinality Estimation (VLDB 2019)

    Naru，NeuroCard的前序工作，使用Masked Autoencoder (MADE)

    https://github.com/naru-project/naru

  • NeuroCard: One Cardinality Estimator for All Tables (VLDB 2020)

    https://github.com/neurocard/neurocard

  • Deep Learning Models for Selectivity Estimation of Multi-Attribute Queries (SIGMOD 2020)

  • Learning to Sample: Counting with Complex Queries (VLDB 2020)

  • Astrid: Accurate Selectivity Estimation for String Predicates using Deep Learning (VLDB 2021)

  • Learned Cardinalities: Estimating Correlated Joins with Deep Learning （CIDR2019）

    MSCN

    https://github.com/andreaskipf/learnedcardinalities

  • Consistent and Flexible Selectivity Estimation for High-Dimensional Data (SIGMOD 2021)

    混合模型

  • Learning State Representations for Query Optimization with Deep Reinforcement Learning (DEEM@SIGMOD2018)

  • An Empirical Analysis of Deep Learning for Cardinality Estimation (arXiv 2019)

    使用NN和RNN进行基数估计

  • Efficiently Approximating Selectivity Functions using Low Overhead Regression Models (VLDB 2020)

    微软的工作，提出了一种新的模型构建方法，该方法增量生成训练数据并使用近似选择性标签，将总构建成本降低一个数量级

  • Flow-Loss: Learning Cardinality Estimates That Matter (arXiv 2020)

    针对基数估计问题引入一个新的损失函数flow-loss

  • Flexible Operator Embeddings via Deep Learning (arXiv 2019)

5. Query Plan

前四篇为同一作者Ryan Marcus的工作：

  • Towards a Hands-Free Query Optimizer through Deep Learning （CIDR 2019)

    这个作者也是后来Neo、Bao的作者，“愿景论文”，不涉及代码。

  • Neo: A Learned Query Optimizer (VLDB 2019)

    论文中仅开源了行向量嵌入部分的代码：https://github.com/parimarjan/db-embedding-tools

  • Bao: Learning to Steer Query Optimizers (2020)

  • Bao: Making Learned Query Optimization Practical (SIGMOD 2021)

    https://github.com/learnedsystems/baoforpostgresql

    2020和2021这两篇基本一样，可以只看2021 SIGMOD

微软：

  • Microlearner: A fine-grained Learning Optimizer for Big Data Workloads at Microsoft (ICDE 2021)

    微软的工作，偏向微软实际应用，描述了为 Microsoft 的大数据工作负载构建学习查询优化器。

  • Steering Query Optimizers: A Practical Take on Big Data Workloads (SIGMOD 2021)

其他：

  • ML-based Cross-Platform Query Optimization (ICDE 2020)

    文章基于的跨平台系统Rheem：https://github.com/rheem-ecosystem/rheem

    Rheem上的工作负载：https://github.com/rheem-ecosystem/rheem-benchmark

    其余源码未找到

  • Learning State Representations for Query Optimization with Deep Reinforcement Learning (DEEM@SIGMOD2018)

    文章中重点在子查询的特征编码和状态转移函数的构建，加上发表年份较早(18年)，更多的像是对机器学习应用到查询优化器的探索，可略读
